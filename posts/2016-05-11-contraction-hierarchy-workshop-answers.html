Exersise 5

  1. C is settled. D, E, F are added to the queue with priorities 1, 1, and 2. A is skipped as it of a lower contraction order.
  2. Either E or D could be visited next depending on the queue, as they are the same distance. We will Assume D is settled next. Edges to E and F are followed but do not result in an improvement.
  3. E is visited. The edge to F is followed, but does not result in an improvement.
  4. F is visited. The edge to B is not followed, as B is of a lower contraction order than F.
  5. The algorithm terminates outputting C: 0, D: 1, E: 1, F: 2.

Exersise 2 The path u-v-w must be longer than either u-c-w or some other path from u to w as a shotcut was not created from u to w when c was contracted. So considering u-c-w again would not affect the result.

For the case where c represents multiple nodes, the contraction process ensures that if there was a shortest path, P, through c, after the contraction shortcuts would have been created so that there would be a path from u to w of the same length as P. Once again, considering the path that goes through c does not affect the result.

Exersise 3 A Dijkstra search from u to w will consider the path u-v-w. As the length of u-v-w equals the length of u-v-w no shortcuts will ever be created. Note: This issue can be incorrectly fixed by requiring witness paths to be strictly shorter than the u-v-w. This will make the algorithm work, but the contracted graph will be denser than it could be.
